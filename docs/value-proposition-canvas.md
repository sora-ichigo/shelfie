## Customer Profile（顧客プロファイル）

### Customer Jobs（顧客が達成したいこと）

**機能的ジョブ**

- 自分が持っている本・読んだ本を把握したい
- 読みたい本を忘れないように記録しておきたい
- 今読んでいる本の進捗を管理したい
- SNS で見かけて気になった本をすぐに登録したい
- テーマやジャンルごとに本を整理したい

**感情的ジョブ**

- 自分だけの本棚を眺めて満足感を得たい
- 読了した本の蓄積を実感したい
- 「本を読む自分」を肯定し、読書のモチベーションを保ちたい

**社会的ジョブ**

- 読んだ本やおすすめを友人・フォロワーにシェアしたい（将来）
- 自分の読書スタイルを表現したい（将来）

---

### Pains（ペイン）

**本の管理に関するペイン**

- 「あの本読んだっけ？」と購入済み・既読の本を忘れてしまう
- 積読が増えるが、何が積まれているか把握できない
- SNS で気になった本を見つけても、後で思い出せない

**既存アプリに対するペイン**

- **UI/UX が古い・もっさりしている** — 主要アプリ（ブクログ・読書メーター）は10年以上前からのサービスで、画面遷移が遅い、デザインが時代遅れ
- **機能過多で取っつきにくい** — コミュニティ・談話室・レビューSNS 等の機能が多すぎて、単に本を記録したいだけの人には重い
- **AI・最新技術への対応が遅い** — 海外では StoryGraph が AI レコメンデーション・ムードベース検索を実装している一方、国内主要アプリは AI 活用がほぼゼロ
- **開発が停滞気味** — 長期運営の国内アプリは大きな機能更新が少なく、モダンな UX トレンド（ダークモード、アニメーション、ジェスチャー操作等）に追従していない
- **プラットフォーム制約** — ビブリアは iOS 専用、読書メーターはデータエクスポート不可など、各アプリに構造的制約がある

---

### Gains（ゲイン）

- すべての蔵書をいつでもどこでも確認できる
- バーコードをかざすだけで本を登録できる手軽さ
- 読書状況（積読 / 読書中 / 読了 / 読まない）が一目でわかる
- お気に入りリストで本を自由にグルーピングできる
- 読了した本に評価やメモを残せる
- 使っていて心地よい、モダンで洗練された操作体験

## 既存の代替手段と課題

### 国内主要アプリ

| アプリ | ユーザー規模 | 強み | 課題・ペイン |
|---|---|---|---|
| **ブクログ** | 200万人+ | 管理機能が充実、タグ分類、新刊ニュース、CSV エクスポート | UI が古い、SNS 機能弱い、ソート機能不足、Web サービス発の設計 |
| **読書メーター** | 国内最大級 | コミュニティが活発、レビュー数が多い | 画面遷移が遅い、5段階評価なし、データエクスポート不可、並べ替え不可 |
| **ビブリア** | - | シンプルで見やすい、ソート充実 | iOS 専用、カテゴリ分類不可、SNS 機能なし |
| **Readee**（楽天） | - | 楽天購入履歴と自動連携 | 楽天エコシステム依存、機能が限定的 |

**共通する構造的課題**:
- いずれも2010年代前半までに確立されたサービスで、UI/UX のモダナイズが遅れている
- AI・機械学習の活用がほぼない（レコメンデーション、自動分類、読書インサイト等）
- 開発リソースが限られ、大きな機能進化が停滞している

### 海外アプリ

| アプリ | 強み | 日本市場での課題 |
|---|---|---|
| **Goodreads**（Amazon） | 世界最大の読書 SNS、圧倒的な書誌データ | 日本語書籍のカバレッジが低い、UI が古い |
| **StoryGraph** | AI レコメンデーション、ムード検索、詳細な統計 | 日本語未対応、日本の書籍データが不足 |
| **Literal Club** | モダン UI、人ベースのレコメンド | 日本語未対応、日本のユーザーベースなし |

**海外アプリの共通課題**: 技術的には先進的だが、日本語書籍・楽天/Amazon.co.jp との連携がなく、日本市場では実用的でない

## Value Map（バリューマップ）

### Products & Services（提供する製品・サービス）

| 機能 | 概要 |
|---|---|
| 本の検索 | キーワード検索で楽天ブックス・Google Books の書籍データを横断検索 |
| ISBN バーコードスキャン | カメラでバーコードを読み取り、瞬時に書籍を特定・登録 |
| 本棚管理 | 読書ステータス別（読書中 / 積読 / 読了 / 読まない）にグリッド表示 |
| ブックリスト | テーマ・ジャンルなど自由なくくりで本をまとめるリスト機能 |
| 評価・メモ | 5段階の星評価と読書メモを本ごとに記録 |
| 本の詳細表示 | 書誌情報の確認、楽天ブックス・Amazon への外部リンク |

---

### Pain Relievers（ペインの解消）

| ペイン | 解消方法 |
|---|---|
| 既読・未読を忘れる | 4つの読書ステータスで管理。本棚画面で一目で把握 |
| 積読が把握できない | 「積読（backlog）」セクションに集約表示 |
| SNS で見た本を忘れる | バーコードスキャンまたは検索で即登録。最短2タップで本棚に追加 |
| 既存アプリの UI が古い・遅い | Flutter × Material 3 でネイティブ級のパフォーマンスとモダン UI を実現。ダークモード標準、滑らかなアニメーション |
| 既存アプリが多機能すぎる | 本棚・検索・リストの3機能に集中したシンプル設計。迷わず使える |
| AI・最新技術への対応がない | モダンな技術スタックで継続的に進化できる基盤。AI レコメンデーション等の拡張余地 |
| プラットフォーム制約 | iOS / Android 両対応。Flutter による一貫した体験 |

---

### Gain Creators（ゲインの創出）

| ゲイン | 実現方法 |
|---|---|
| いつでも蔵書確認 | モバイルアプリで手のひらに本棚を持ち歩ける |
| 手軽な登録体験 | バーコードスキャン + 検索の2つの導線。書誌情報は自動取得 |
| 読書状況の可視化 | ステータス別セクション表示で全体像を把握。セクションごとの冊数表示 |
| 自由なグルーピング | ブックリストで「今年読みたい本」「おすすめSF」など自由にキュレーション |
| 読書の振り返り | 星評価とメモで読後の感想を蓄積。読了数で達成感を可視化 |
| 心地よい操作体験 | 表紙を活かしたビジュアル本棚、スムーズなアニメーション、直感的なジェスチャー操作 |

## Fit（フィット分析）

### コアバリュー

> **「あなたの本棚を、手のひらに」**
>
> 既存の読書アプリが抱える「古い・重い・多すぎる」を解消し、
> 本を読むすべての人にとっての must-have な本棚体験を提供する。

### ターゲットユーザー

**初期フォーカス（ローンチ〜PMF）**

- 年間5冊以上本を読み、蔵書を記録・管理したいと感じている人
- 既存の読書アプリ（ブクログ・読書メーター等）を使ったが、UI の古さや使いにくさに不満を感じている人
- シンプルで洗練されたツールを好む人

**将来ビジョン**

- 10代〜20代を含む、本を読む（または読みたい）すべての人の must-have
- 「読書家」を自称しない層 — SNS で話題の本が気になる、年に数冊は読む、これから読書を始めたい人
- Z 世代の読書行動（SNS で本を発見、紙の本を好む、「エモさ」を重視）にフィットするプロダクト

**広がりの仮説**: 「狭く深く」→「広く深く」の段階的拡大

```
Phase 1: 既存読書アプリに不満を持つ管理意識の高い読書家
         → シンプル × モダン UI で乗り換え需要を獲得
Phase 2: 本は読むが記録していない層
         → バーコードスキャンの手軽さと心地よい UX で習慣化を促進
Phase 3: 本を読みたいと思っている・SNS で本に触れている若年層
         → AI レコメンデーション、ソーシャル機能で「次の一冊」との出会いを提供
```

### 競合との差別化ポイント

| 観点 | 国内既存アプリ | 海外先進アプリ | **Shelfie** |
|---|---|---|---|
| UI/UX | レガシー、画面遷移が遅い | モダンだが日本語非対応 | **モダン × 日本語ネイティブ** |
| 日本語書籍 | 強い（楽天連携等） | 弱い | **楽天 + Google 横断検索** |
| AI 活用 | ほぼなし | StoryGraph がリード | **今後の最大の差別化領域** |
| 開発速度 | 停滞気味 | 活発 | **モダンスタックで高速に進化** |
| ターゲット | 読書家コミュニティ | 英語圏の読書家 | **日本の全読書層** |

### Shelfie が勝てる理由

1. **国内既存アプリへの不満が顕在化している** — UI が古い、遅い、重いという声は明確に存在する
2. **海外の先進アプリが日本市場に入れていない** — 日本語書籍データと検索APIの壁がある
3. **「モダン UI × 日本語書籍 × AI」の組み合わせが空白地帯** — この交差点にいるプレイヤーがいない
4. **技術的に正しいスタックを選択済み** — Flutter, GraphQL, モダン API 設計で継続的な進化が可能

## 未検証の仮説・今後の検討事項

### ターゲット・市場
- [ ] 「既存アプリに不満を持つ読書家」は初期ターゲットとして十分な規模か
- [ ] 10代〜20代の非読書家層のニーズは Phase 3 まで待てるか、早期に取り込むべきか
- [ ] Z 世代の「SNS で本を発見する」行動にどう接続するか（SNS 連携、シェア機能）

### プロダクト
- [ ] AI レコメンデーション（StoryGraph 的な機能）の優先度と実装時期
- [ ] ソーシャル機能（本棚の共有、おすすめ共有）のスコープと優先度
- [ ] 読書統計（月別冊数、ジャンル分布など）のニーズはどの程度あるか
- [ ] 電子書籍との連携（Kindle 既読の自動取り込みなど）は差別化になるか

### ビジネス
- [ ] マネタイズモデル（フリーミアム、サブスクリプション等）の検討
- [ ] 国内既存アプリからの乗り換えを促す具体的な施策（データインポート等）
