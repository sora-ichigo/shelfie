## Customer Profile（顧客プロファイル）

### Customer Jobs（顧客が達成したいこと）

**機能的ジョブ**

- 自分が持っている本・読んだ本を把握したい
- 読みたい本を忘れないように記録しておきたい
- 今読んでいる本の進捗を管理したい
- 気になった本をすぐに登録したい
- テーマやジャンルごとに本を整理したい

**感情的ジョブ**

- 自分だけの本棚を眺めて満足感を得たい
- 読了した本の蓄積を実感したい
- 読書の習慣を維持・可視化したい

**社会的ジョブ**

- (現時点では対象外 — ソーシャル機能は未実装)

---

### Pains（ペイン）

- 物理的な本棚は場所を取り、蔵書が増えると管理しきれない
- 本のタイトルや著者名を手入力するのが面倒
- 「あの本読んだっけ？」と購入済み・既読の本を忘れてしまう
- 積読が増えるが、何が積まれているか把握できない
- 複数の書店サイトで本を探すのが手間
- 読書記録アプリが多機能すぎて使いこなせない

---

### Gains（ゲイン）

- すべての蔵書をいつでもどこでも確認できる
- バーコードをかざすだけで本を登録できる手軽さ
- 読書状況（積読 / 読書中 / 読了 / 読まない）が一目でわかる
- お気に入りリストで本を自由にグルーピングできる
- 読了した本に評価やメモを残せる

## Value Map（バリューマップ）

### Products & Services（提供する製品・サービス）

| 機能 | 概要 |
|---|---|
| 本の検索 | キーワード検索で楽天ブックス・Google Books の書籍データを横断検索 |
| ISBN バーコードスキャン | カメラでバーコードを読み取り、瞬時に書籍を特定・登録 |
| 本棚管理 | 読書ステータス別（読書中 / 積読 / 読了 / 読まない）にグリッド表示 |
| ブックリスト | テーマ・ジャンルなど自由なくくりで本をまとめるリスト機能 |
| 評価・メモ | 5段階の星評価と読書メモを本ごとに記録 |
| 本の詳細表示 | 書誌情報の確認、楽天ブックス・Amazon への外部リンク |

---

### Pain Relievers（ペインの解消）

| ペイン | 解消方法 |
|---|---|
| 蔵書を管理しきれない | デジタル本棚でステータス別に自動分類。ソート・フィルタで即座に検索可能 |
| 手入力が面倒 | ISBN バーコードスキャンでカメラをかざすだけ。書誌情報は自動取得 |
| 既読・未読を忘れる | 4つの読書ステータスで管理。本棚画面で一目で把握 |
| 積読が把握できない | 「積読（backlog）」セクションに集約表示 |
| 複数サイトで探すのが手間 | 楽天ブックス + Google Books を一括検索。1つの検索窓で完結 |
| 多機能すぎて使えない | シンプルな UI 設計。本棚・検索・リストの3機能に集中 |

---

### Gain Creators（ゲインの創出）

| ゲイン | 実現方法 |
|---|---|
| いつでも蔵書確認 | モバイルアプリで手のひらに本棚を持ち歩ける |
| 手軽な登録体験 | バーコードスキャン + 検索の2つの導線。最短2タップで本棚に追加 |
| 読書状況の可視化 | ステータス別セクション表示で全体像を把握。セクションごとの冊数表示 |
| 自由なグルーピング | ブックリストで「今年読みたい本」「おすすめSF」など自由にキュレーション |
| 読書の振り返り | 星評価とメモで読後の感想を蓄積。読了数で達成感を可視化 |

## Fit（フィット分析）

### コアバリュー

> **「あなたの本棚を、手のひらに」**
>
> バーコードスキャンによる手軽な登録と、ステータス別表示によるシンプルな蔵書管理。
> 読書家が「自分の本棚」を持ち歩けるミニマルな体験を提供する。

### ターゲットユーザー

- 年間10冊以上読む読書家
- 蔵書が増えてきて管理に困り始めた人
- シンプルなツールを好む人（SNS 的な機能より管理機能を重視）

### 競合との差別化ポイント

| 観点 | Shelfie のポジション |
|---|---|
| 機能の幅 | ミニマル — 蔵書管理に特化。レビューSNS・コミュニティ機能は持たない |
| 登録体験 | バーコードスキャン + 楽天/Google 横断検索で日本語書籍に強い |
| UI/UX | シンプル・ダークモード。本の表紙を活かしたビジュアル表示 |
